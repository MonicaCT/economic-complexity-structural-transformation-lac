# Recruiter Guide

## What this project demonstrates

This repository is designed to show the ability to turn a large public trade-data workflow into decision-ready analytical products. It combines reproducible research, data validation, indicator engineering, dashboard communication and policy-oriented interpretation.

## Best evidence for data analyst roles

| Skill | Evidence in repository |
|---|---|
| Data processing | Large country-product-year analytical panel and validation outputs. |
| Data-quality validation | `data_integrity_summary.csv`, ECI/PCI validation summaries and repository checks. |
| Indicator engineering | RCA, ECI, PCI, diversity, HHI, entropy, density and opportunity metrics. |
| Data storytelling | README, executive tables, policy brief, static website and dashboard screenshots. |
| Dashboard thinking | Shiny app, screenshot gallery and interactive static explorer. |
| Reproducibility | Documented workflow, methodology, validation reports and citation metadata. |
| Domain expertise | Economic complexity, structural transformation, export diversification and Latin America. |

## Best evidence for research data analyst roles

- Methodological documentation clarifies assumptions and limitations.
- Fixed-effects models are interpreted as observational associations.
- The Product Space is documented as an analytical network, not merely a visual.
- Data-quality warnings are visible and not hidden.

## Suggested review path

1. Start with `README.md` for the portfolio snapshot and main findings.
2. Open `docs/index.html` for the static public-facing gallery.
3. Open `docs/explore/index.html` for interactive filtering and recruiter-facing data exploration.
4. Review `docs/EXECUTIVE_SUMMARY.md` and `docs/KPI_DICTIONARY.md`.
5. Review `docs/DATA_QUALITY_REPORT.md` and `docs/DATA_MODEL.md` for technical depth.
6. Review the paper and policy brief for scientific and stakeholder communication.

## What not to infer

- Product opportunity scores are not direct policy recommendations.
- Fixed-effects estimates are not causal effects.
- Static website data are curated slices of existing final outputs, not a replacement for the full analytical pipeline.
